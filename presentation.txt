EXECUTIVE SUMMARY & SYSTEM VISION
---------------------------------
To address the Traffic Demand Prediction challenge, we designed an end-to-end AI 
system that provides valuable insights into passenger travel patterns, booking 
behavior, and traffic dynamics. By accurately forecasting future traffic demand 
and profiling spatial-temporal fluctuations, this system enables data-driven 
strategies to optimize logistics, alleviate urban traffic congestion, and enhance 
resource dispatching—core operational goals for Flipkart's supply chain networks.

Our predictive engine is a blended ensemble of state-of-the-art Gradient Boosted 
Decision Trees (GBDTs)—specifically CatBoost, LightGBM, and XGBoost—built to 
maximize the Scaled R2 score metric under tight temporal and spatial constraints.


1. SYSTEM ALIGNMENT WITH THE PROBLEM TASK
-----------------------------------------
The task requires designing a system that uncovers passenger travel patterns and 
predicts demand in the travel industry. Our solution achieves this through a 
dual-purpose architecture:

* Demand Prediction Engine:
  - Optimizes the competition's evaluation metric: 
    Score = max(0, 100 * R2_Score)
  - Successfully handles the 77,299 x 11 training instances to forecast demand 
    for the 41,778 x 10 test entries.

* Spatial-Temporal Insights Module:
  - Booking & Travel Rhythms: Analyzes demand fluctuations over different days 
    (Day 48 vs. 49) and hourly timelines (morning and evening rush hour peaks).
  - Infrastructure Context: Leverages road layout (RoadType, NumberofLanes) and 
    landmark locations to model spatial passenger booking behavior.
  - Environmental Impact: Incorporates temperature and weather conditions (e.g., 
    Sunny vs. Snowy demand shifts) to capture real-time weather-responsive booking patterns.


2. THE DATASET & DYNAMIC VARIABLES UTILIZED
-------------------------------------------
Our system ingests and processes all raw variables provided in the datasets:
* Index: Maintained as the unique identifier for compliance with target matching.
* geohash: Decoded into geographic coordinates and hierarchical substrings to 
  capture localized passenger demand patterns.
* day: Modeled to capture day-to-day progression and weekly patterns (weekday vs. weekend).
* timestamp: Parsed to capture continuous diurnal patterns and rush hour spikes.
* RoadType, NumberofLanes, LargeVehicles: Modeled to represent infrastructure 
  capacity (e.g., Highway and multi-lane roads correlate with higher traffic volume).
* Temperature, Weather: Evaluated to understand meteorological impacts on travel behavior.
* demand: Scaled using log-transformation to stabilize the regression target.


3. HIGH-IMPACT FEATURE ENGINEERING
----------------------------------
Our feature pipeline converts raw variables into high-fidelity predictive features:

* Spatial Representation & Hierarchical Fallbacks (geohash):
  - Coordinate Transformation: Decoded raw `geohash` strings into floating-point 
    `Latitude` and `Longitude` using geodesic mapping, transforming arbitrary 
    identifiers into metric space.
  - Spatial Scale Truncation: Generated prefix substrings of lengths 1 to 5 
    (`geohash_1` to `geohash_5`). This structures space hierarchically, allowing 
    the tree algorithms to learn coarse-grained regional profiles down to 
    micro-local traffic zones. Crucially, this resolves the "cold-start" problem 
    for the 10 geohash locations present in the test set but absent in training.

* Temporal Rhythms & Cyclical Math (timestamp & day):
  - Cyclical Encoding: Projected time of day ('TimeInMinutes') onto a 2D 
    unit circle using trigonometric transformations:
      sin_TimeInMinutes = sin(2 * pi * TimeInMinutes / 1440.0)
      cos_TimeInMinutes = cos(2 * pi * TimeInMinutes / 1440.0)
    This ensures that 23:59 and 00:00 are represented as chronologically close, 
    eliminating the artificial boundary discontinuity.
  - Weekly Profiles: Extracted day of the week ('day % 7') and weekend indicator 
    flags to model changes in passenger corporate vs. leisure booking behavior.
  - Peak-Traffic Heuristics: Flagged morning (08:00 - 10:00) and evening 
    (16:00 - 19:00) rush-hour slots.

* Target Stabilization (demand):
  - The target variable (`demand`) exhibits a severe right skew (~3.73). Training 
    directly on raw demand leads to model bias towards low-demand periods. 
    Applying a log-transform `np.log1p(demand)` normalizes the target skewness 
    to `-0.728`, stabilizing the GBDT split calculations and directly boosting 
    the Scaled R2 metric.


4. CORE MODEL ARCHITECTURE & BLENDED ENSEMBLE
----------------------------------------------
Rather than relying on a single estimator, our final architecture blends three 
diverse gradient-boosting frameworks:

* CatBoost Regressor: Excellent at handling raw categorical features natively 
  without overfitting, using symmetric trees.
* LightGBM Regressor: Extremely fast, leaf-wise tree growth that captures deep 
  non-linear interactions in high-cardinality features (like geohashes).
* XGBoost Regressor: Histogram-optimized tree growth with categorical support, 
  offering a diverse gradient boosting regularization dynamic.

Ensemble Blending:
Predictions from all three models are combined via a simple average. This acts as 
a powerful regularizer, smoothing out individual model errors and maximizing 
generalization on the unseen test set. Predictions are mathematically clipped to 
the valid domain boundaries [0.0, 1.0].

6. REPLICATION & EXECUTION GUIDE (HOW TO RUN)
---------------------------------------------
Follow these steps in your local environment to replicate the final submission:

* Step 1: Place the dataset files inside the `dataset/` directory:
  - `dataset/train.csv` (77,299 rows)
  - `dataset/test.csv` (41,778 rows)
  - `dataset/sample_submission.csv` (5 rows template)

* Step 2: Install dependencies using the pip command shown in Section 5.

* Step 3: Train the Machine Learning Ensemble:
  ```bash
  python train_model.py
  ```
  This trains the CatBoost, LightGBM, and XGBoost models on a 5-fold cross-validation scheme, performs out-of-fold blending, and outputs raw predictions to `submission.csv` in the root directory.

* Step 4: Run the Post-Processing & Calibration:
  ```bash
  python thany/traffic_demand_solution.py
  ```
  This script will automatically detect the GBDT predictions in `submission.csv`, clip the values to the valid [0.0, 1.0] range, and generate the final submission file: **`thany/traffic_demand_submission.csv`**. It also generates the Jupyter notebook `thany/traffic_demand_prediction.ipynb` and `thany/validation_report.json`.

* Step 5: (Optional) Run the EDA Generator:
  ```bash
  python build_eda.py
  ```
  This compiles and executes the Exploratory Data Analysis notebook `eda.ipynb`, rendering all feature correlation, missing value, and demand timeline plots.
